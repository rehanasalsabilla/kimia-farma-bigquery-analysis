# 📊 Project Overview - Kimia Farma Business Performance Analysis

## 📌 Background
Kimia Farma’s sales performance during the 2020–2023 period shows a relatively fluctuating pattern but tends to be stable overall. This indicates variations in performance across time, regions, and branches. In addition, there is a potential gap between branch service quality and customer transaction experience, as well as the impact of discount strategies on profitability.

---

## 🎯 Objectives
The objectives of this project are:
- Analyze sales and profit trends for the 2020–2023 period  
- Identify top-performing branches and provinces  
- Identify gaps between branch ratings and transaction ratings  
- Evaluate the impact of discounts on profit  
- Provide insights to support business decision-making  

---

## 📂 Dataset
This project uses three main datasets:
- **Transaction Data (`kf_final_transaction`)**: contains sales, profit, discount, and transaction rating  
- **Product Data (`kf_product`)**: contains product name, category, and price  
- **Branch Data (`kf_kantor_cabang`)**: contains branch information such as location and rating  

These datasets are combined into an analytical table (`kf_analysis`) for dashboard analysis.

---

## 🛠️ Tools & Technologies
- **BigQuery** → Data processing and transformation  
- **SQL** → Data joining, calculation, and analysis  
- **Looker Studio** → Dashboard visualization  

---

## 🔄 Project Workflow
This project follows an end-to-end data analysis process:

1. **Data Ingestion** → Import dataset into BigQuery  
2. **Data Understanding** → Understand structure, columns, and relationships  
3. **Data Processing** → Perform joins and calculations using SQL  
4. **Data Transformation** → Create analytical table (`kf_analysis`)  
5. **Data Visualization** → Build interactive dashboard in Looker Studio  
6. **Reporting** → Generate insights and business recommendations

This project focuses on analyzing:
